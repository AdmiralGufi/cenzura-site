# CENZURA — сайт-витрина бренда

Минималистичный сайт бренда CENZURA на **Next.js 14 + Tailwind CSS** с интеграцией каталога **Wildberries**. Покупка совершается на WB — сайт служит витриной: каталог, страницы товаров, история бренда, контакты.

## Быстрый старт

```bash
npm install
npm run sync     # подтянуть реальные товары с Wildberries (см. ниже)
npm run dev      # http://localhost:3000
```

Без `npm run sync` сайт работает на демо-товарах из `data/products.json`.

## Как работает интеграция с Wildberries

Два уровня (от надёжного к запасному):

1. **`npm run sync`** — скрипт `scripts/sync-wb.mjs` обновляет `data/products.json`:
   - с токеном в `.env` → через **официальный Content API продавца** (названия, описания, фото, размеры, категории) + **API цен**;
   - без токена → через публичный JSON-API каталога бренда (работает только с IP РФ/СНГ); из него же добавляются рейтинг и отзывы.
2. **Живой каталог на лету** — `lib/wb.ts` при каждой пересборке страницы (ISR, кэш 1 час) пробует публичный API; если он недоступен (например, зарубежный хостинг), используется снапшот `data/products.json`.

Рекомендуемый процесс: запускайте `npm run sync` при изменении ассортимента (или по расписанию через GitHub Actions), коммитьте `data/products.json` — хостинг пересоберёт сайт.

## Токен WB API

- Лежит в `.env` (`WB_API_TOKEN=…`), файл в `.gitignore` — **не коммитить, не публиковать**.
- Используется только скриптом sync, в браузер не попадает.
- Текущий токен действует до **~10.12.2026**. Перевыпуск: ЛК WB → Профиль → Настройки → Доступ к API (категории «Контент» и «Цены и скидки»).
- Если токен мог утечь — перевыпустите его там же.

## Настройка под себя

Все «ручки» в одном файле — `lib/config.ts`:

- `url` — домен сайта после деплоя (важно для SEO/sitemap);
- `wildberries.domain` — `wildberries.ru` или `wildberries.kg` (куда ведут кнопки «Купить»);
- `contacts` — Instagram, Telegram, email.

Контент страницы «О бренде» — `app/about/page.tsx`. Демо-товары — `data/products.json`.

## Форма обратной связи

Работает без бэкенда через [FormSubmit](https://formsubmit.co): письма приходят на email из `lib/config.ts`. **Первая отправка** пришлёт на этот адрес письмо с подтверждением — нажмите в нём кнопку, после этого форма активна.

## Деплой

### Vercel (просто)
1. Запушьте проект на GitHub (убедитесь, что `.env` не попал в репозиторий).
2. [vercel.com](https://vercel.com) → Add New Project → импорт репозитория → Deploy (настройки по умолчанию).
3. После деплоя впишите выданный домен в `lib/config.ts` → `url` и запушьте ещё раз.

Примечание: с серверов Vercel (США) публичный API WB обычно недоступен, поэтому сайт будет показывать данные из `data/products.json` — это штатный режим. Обновление каталога = `npm run sync` + push.

### Хостинг в РФ/СНГ (Timeweb, рег.ру и т.п.)
Подойдёт любой Node-хостинг: `npm run build && npm run start` (порт 3000). Бонус: оттуда работает и «живой» каталог без sync.

### Автообновление каталога (опционально)
GitHub Actions по расписанию: добавьте секрет `WB_API_TOKEN` в настройках репозитория и workflow, который запускает `npm run sync` и коммитит изменённый `data/products.json`.

## Структура

```
app/            страницы (App Router): главная, catalog, product/[id], about, contacts
  sitemap.ts    карта сайта (+ все товары)
  robots.ts
components/     Header, Footer, ProductCard, CatalogClient (фильтры/сортировка/поиск), Gallery
lib/            config.ts (настройки), wb.ts (данные WB), format.ts, types.ts
data/           products.json — снапшот каталога
scripts/        sync-wb.mjs — синхронизация с WB
public/demo/    SVG-заглушки демо-товаров
```

## Что уже включено

- Адаптивный mobile-first дизайн, супер-минимализм (чёрный/белый/беж)
- Каталог: фильтр по категориям, сортировка (новинки/цена), поиск, «Показать ещё»
- Страница товара: галерея, размеры, цвета, описание, кнопка «Купить на Wildberries»
- SEO: мета-теги, Open Graph, sitemap.xml, robots.txt, JSON-LD (Product) на товарах
- ISR-кэширование запросов к WB (1 час)
